# Indicators Outputs

This document describes the artifacts emitted by `examples/stack/indicators/`.

This stage mirrors the role of `examples/indicators/`, but for the practitioner stack the deliverable is a lightweight progression viewer rather than a full indicator onboarding pipeline.

## Stage 0 Contract

The final implementation of this stage is expected to emit:

- a curated image progression manifest
- indicator progression payloads
- image-to-indicator pairing artifacts

## Expected Consumers

- the `examples/stack/indicators/` UI

## Verification Gate

Before this stage is treated as stable, it must be manually verified in the UI for:

- synchronized image and indicator stepping
- readable similarity and restoration-progress display

This file will be updated after Stage 6 with exact filenames and contracts.
