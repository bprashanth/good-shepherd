# Nursery Manager: OpenClaw Installation

This guide documents the complete setup process for OpenClaw in the nursery assistant environment.

## Prerequisites

- Docker and Docker Compose installed
- Git installed
- Access to the good-shepherd repository
- Claude API key or Anthropic API key

---

## Installation Steps

### 1. Clone OpenClaw Repository

First, clone the official OpenClaw repository to build the image:

```bash
# Clone to a temporary location
cd /tmp
git clone https://github.com/openclaw/openclaw.git
cd openclaw
```

### 2. Build the OpenClaw Docker Image

Build the image that will be used in our nursery setup:

```bash
docker build -t openclaw:local -f Dockerfile .
```

**Expected output:** `Successfully tagged openclaw:local`

### 3. Set Up Configuration Directory

Navigate to the nursery assistant directory and create the OpenClaw config:

```bash
cd /home/desinotorious/src/github.com/bprashanth/good-shepherd/examples/stacks/community/assistant

# Create .openclaw directory if it doesn't exist
mkdir -p .openclaw
```

### 4. Initial Onboarding

Run the interactive onboarding wizard to configure OpenClaw:

```bash
docker compose run --rm openclaw-cli onboard
```

**Prompts you'll see:**
- **Gateway binding**: Choose `lan` (allows access from local network)
- **Auth method**: Choose `token` (secure token-based auth)
- **Gateway token**: Auto-generated (will be saved to .env)
- **Tailscale**: Choose `Off` (not needed for local setup)
- **Install daemon**: Choose `No` (we're using docker compose)

**For WhatsApp integration:**
- When prompted, enable WhatsApp
- Scan the QR code with WhatsApp mobile app
- WhatsApp session will be saved to `.openclaw/credentials/`

### 5. Configure the Control UI

Edit `.openclaw/openclaw.json` to enable the web interface:

```bash
nano .openclaw/openclaw.json
```

Ensure the `gateway` section includes:

```json
"gateway": {
  "port": 18789,
  "bind": "lan",
  "controlUi": {
    "enabled": true,
    "allowInsecureAuth": true
  },
  "auth": {
    "mode": "token",
    "token": "<your-generated-token>"
  }
}
```

**Security note:** `allowInsecureAuth: true` is only for local development. For production, use proper authentication.

### 6. Verify .env File

Check that `.env` was created with correct paths:

```bash
cat .env
```

Should contain:
```bash
OPENCLAW_CONFIG_DIR=./.openclaw
OPENCLAW_WORKSPACE_DIR=./.openclaw/workspace  # Note: fixed typo
OPENCLAW_GATEWAY_PORT=18789
OPENCLAW_BRIDGE_PORT=18790
OPENCLAW_GATEWAY_BIND=lan
OPENCLAW_GATEWAY_TOKEN=<auto-generated-token>
OPENCLAW_IMAGE=openclaw:local
```


### 7. Fix Permissions (if needed)

If you copied config from `~/.openclaw/`, fix permissions for uid 1000 (node user):

```bash
sudo chown -R 1000:1000 .openclaw
```

### 8. Start the Services

Start the OpenClaw gateway:

```bash
docker compose up -d openclaw-gateway
```

**Verify it's running:**
```bash
docker compose ps
```

Should show `openclaw-gateway` as `Up`

### 9. Access the Web Interface

Open your browser and navigate to:
```
http://localhost:18789/chat
```

You should see the OpenClaw control UI.

### 10. Test the Setup

**Option A: Via CLI (in container)**
```bash
docker compose exec openclaw-gateway node dist/index.js agent --message "hi" --agent main --local

# This spins up a container called openclaw-cli defined in the compose vs the previous command uses the index.js in the existing container 
docker compose run --rm openclaw-cli --help
```

**Option B: Via WhatsApp**
If you scanned the QR code, send a message to yourself on WhatsApp. The bot should respond.

**Option C: Via Web UI**
Use the chat interface at `http://localhost:18789/chat`

## Troubleshooting

### Container won't start
```bash
# Check logs
docker compose logs openclaw-gateway --follow

# Common issues:
# - Port 18789 already in use
# - .openclaw/ permissions incorrect (needs uid 1000)
# - .env OPENCLAW_CONFIG_DIR path wrong
```

### WhatsApp not working
```bash
# Re-run onboarding to rescan QR
docker compose run --rm openclaw-cli onboard

# Check credentials
ls -la .openclaw/credentials/
```

### Permission denied errors
```bash
# Fix ownership for node user (uid 1000)
sudo chown -R 1000:1000 .openclaw
```

### Web UI shows 401 Unauthorized
- Check `.openclaw/openclaw.json` has `"allowInsecureAuth": true`
- Restart gateway: `docker compose restart openclaw-gateway`

## Security Considerations

### Files That Contain Secrets (NEVER COMMIT)

The following files/directories contain sensitive data:
- `.env` - Gateway tokens
- `.openclaw/openclaw.json` - Auth tokens, API keys
- `.openclaw/credentials/` - WhatsApp session data
- `.openclaw/agents/` - Agent session data

### Verification

Check that `.gitignore` and `.clawdignore` properly exclude:
```bash
# Verify git ignores sensitive files
git status | grep -E ".env|openclaw.json|.openclaw"
# Should show: nothing (all ignored)

# .gitignore should contain:
.openclaw/
.env
*.env
*.bak

# .clawdignore should contain:
.openclaw/openclaw.json
.openclaw/credentials/
.openclaw/agents/
```

**If openclaw.json is tracked by git:**
```bash
# Remove from git tracking (keeps local file)
git rm --cached .openclaw/openclaw.json
git commit -m "Remove sensitive openclaw.json from tracking"
```

## Configuration Reference

### Environment Variables (.env)

| Variable | Description | Default |
|----------|-------------|---------|
| `OPENCLAW_CONFIG_DIR` | Path to config directory | `./.openclaw` |
| `OPENCLAW_WORKSPACE_DIR` | Path to workspace | `./.openclaw/workspace` |
| `OPENCLAW_GATEWAY_PORT` | Web UI port | `18789` |
| `OPENCLAW_BRIDGE_PORT` | Bridge port | `18790` |
| `OPENCLAW_GATEWAY_BIND` | Binding mode | `lan` |
| `OPENCLAW_GATEWAY_TOKEN` | Auth token | Auto-generated |
| `OPENCLAW_IMAGE` | Docker image name | `openclaw:local` |

### Volume Mounts

```yaml
volumes:
  - ./.openclaw:/home/node/.openclaw  # Config and credentials
  # Workspace mount is commented out in docker-compose.yml
  # Uncomment if you need persistent workspace:
  # - ./.openclaw/workspace:/home/node/.openclaw/workspace
```

**Why workspace mount is commented:**
- Gateway doesn't need workspace persistence
- CLI service has it enabled
- Uncomment if you need to access workspace files from host

## Moving to a Different Repository

To replicate this setup elsewhere:

1. **Copy required files:**
   ```bash
   cp docker-compose.yml /path/to/new/repo/
   cp .env /path/to/new/repo/
   ```

2. **Do NOT copy:**
   - `.openclaw/` directory (contains secrets!)
   - Instead, re-run onboarding in new location

3. **Verify:**
   - `openclaw:local` image exists (`docker images | grep openclaw`)
   - `.env` paths are correct for new location
   - Permissions are correct (uid 1000)
